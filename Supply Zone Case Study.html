Background

The brief

This project involved redesigning an existing internal system for supply-zone analysis. While the original system supported the core analysis tasks, it was difficult to use in practice. The goal was to improve usability and clarity while preserving the original analytical workflow - not to reinvent what the tool did, but to make it feel coherent to the people who relied on it every day.

Project overview

Challenge, solution, outcome

The challenge

Fragmented upload and processing steps required frequent page switching - creating high cognitive load and an interrupted user journey from the moment a task began.

The solution

Restructured the information architecture to consolidate the core workflow into a single-page interface, while revamping the UI layout to split complex data charts and reduce visual noise.

The outcome

A streamlined, single-stream workflow that removed redundant navigation, clarified the data charts, and let users move from upload to results without losing context - validated against the friction points surfaced in testing.

User journey

Mapping the file-processing flow

Before touching the interface, I mapped the end-to-end journey of processing a file - from entering the platform to checking analysis results - to locate exactly where the experience broke down.

01Initial evaluation

A heuristic pass to surface the friction

I started with a heuristic evaluation to identify potential friction points in the existing information hierarchy and workflow logic - pairing each observation with a screenshot from the original system.

02Rapid UI fix

A testable baseline

Based on initial observations, I addressed basic visual inconsistencies and immediate usability glitches to create a testable baseline - something concrete to put in front of users.

03Usability testing

Testing the baseline with real users

A prototype incorporating the rapid UI fixes was tested with users to validate the initial assumptions and to deeply understand their interaction challenges.

04Redesign - IA & workflow

Reworking the information architecture

I created a new information architecture and workflow based on the usability-testing findings - auditing the original structure to decide what to keep, merge, or remove, then rebuilding around clearer task initiation and a process-now / save-for-later decision point.

Outcome

One coherent workflow, end to end

By consolidating upload and processing into a single page, standardizing the iconography, and rebuilding the analytics views around progressive disclosure, the redesign turned a capable-but-confusing tool into one that supports the analyst's real flow - from entering the platform to reading the results.
